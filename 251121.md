알고리즘으로 PS 하는 것보다는 성능 분석하는 게 훨씬 재밌다. 

예를 들면, 오늘 list comprehension으로 짠 코드가 있었다. Claude에게 코드 리뷰를 받아보니 조금 다르다. 로직은 거의 똑같아 보였는데, 실행해보니 메모리 사용량에서 거의 6MB 차이가 났다. 어디서 이런 차이가 났을까 궁금해졌다. 실행 시간은 똑같은데, 어떻게 메모리 사용량만 줄였을까. 

확인해보니 generator expression을 사용하면 데이터 전체를 메모리에 담아두고 본 작업에 들어가는 대신 하나씩 가져와서 처리할 수 있다고 한다. 이 문제에서 최대 1만 개의 문자열이 입력되는 걸 감안하면, 6MB의 메모리 차이가 설명된다. 

이런 메커니즘을 추적하면서 성능을 분석하는 건 재밌는 일이다.
